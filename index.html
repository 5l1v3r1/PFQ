PFQ

PFQ: Accelerated Packet Capture Engine for Multi-core Architectures

Download this project as a .zip file Download this project as a tar.gz file

What is PFQ?

PFQ is a novel network-capture engine designed for the Linux Kernel 3.x and modern 64-bit architectures. It is highly optimized for multi-core processors, as well as for network devices supporting multiple hardware queues.

PFQ works with vanilla drivers and provides a script (pfq-omatic) that patches and compiles source codes on-the-fly to obtain accelerated drivers.

Its light weight lock-less architecture is based on the following features:

  • wait-free queuing mechanisms to handle packet dispatching in an ultra-efficient manner.
  • cache-friendly handling for memory allocations/deallocations.
  • hashing algorithms for packets steering across a variable number of user-space sockets.

Features

  • 10-Gbit Line-rate (14,8Mpps) with Intel ixgbe vanilla driver.
  • Socket groups allow for concurrent monitoring of multi-threaded applications.
  • Classification by classes allow for fine-control over packet-steering within groups.
  • Per-group Berkeley filters and per-group vlan filters.
  • Extensible framework engine for in-kernel functional monitoring (continuation passing style).
  • Native bindings for C, C++11 and Haskell languages.
  • PFQ-omatic, a script that allows to compile drivers and get the accelerated versions.
  • Accelerated pcap library.

Performance

PFQ performance is highly dependent on the hardware used.

Running on top of the 2.66Ghz 6-core Xeon equipped and the Intel 82599 10G controller, PFQ can steer to user-space about 14,8 Million packets per second deploying 3 kernel threads.

Details of performance comparison are available here (old version): http://netgroup.iet.unipi.it/software/pfq/performance.html

Source code

The package provides the source code of the PFQ kernel module, user-space libraries for C, C++11 and Haskell languages and a set of diagnostic tools.

Gcc/g++ 4.6 (or higher), the Haskell platform, Cmake and make are required for the compilation.
