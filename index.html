PFQ

PFQ: Accelerated Packet Capture Engine for Multi-core Architectures

View project onGitHub

What is PFQ?

PFQ is a novel network-capture engine designed for the Linux Kernel 3.x and modern 64-bit architectures. It is highly optimized for multi-core processors, as well as for network devices supporting multiple hardware queues.

PFQ works with both vanilla and PFQ-aware drivers (distributed with the package).

Its light weight lock-less architecture is based on the following features:

  • wait-free queuing mechanisms to handle packet dispatching in a ultra-efficient manner.
  • cache-friendly handling for memory allocations/deallocations.
  • hashing algorithms for packets steering across a variable number of user-space sockets (under development).

Performance

PFQ performance is highly dependent on the hardware used.

For instance, running on top of the 2.66Ghz 6-core Xeon equipped and the Intel 82599 10G controller, PFQ can steer to user-space about 13 Million packets per second deploying 12 threads with CPUs load well under 10% each (~40Mpps with full copies).

Details of performance comparison are available here: http://netgroup.iet.unipi.it/software/pfq/performance.html

Source code

The package provides the source code of the Linux Kernel module, some optimized drivers for Intel 82599 NICs, user-space libraries for both C++11 and C language and a set of diagnostic tools.

Gcc/g++ 4.6 (or higher) and cmake are required for the compilation.
